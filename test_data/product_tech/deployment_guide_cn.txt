部署手册 - 产品技术库

第一章 环境准备

1.1 系统要求
操作系统：Ubuntu 20.04 LTS 或 CentOS 8
内存：最低 8GB，推荐 16GB
磁盘空间：最低 100GB SSD
网络：千兆以太网，公网IP

1.2 依赖安装
需要安装以下软件：
- Docker 20.10+
- Docker Compose 2.0+
- Nginx 1.20+
- PostgreSQL 14+
- Redis 6.2+
- Node.js 18+
- Python 3.10+

1.3 网络配置
开放以下端口：
- 80/443：HTTP/HTTPS 服务
- 5432：PostgreSQL 数据库
- 6379：Redis 缓存
- 8080：应用服务
- 9090：监控服务

第二章 应用部署

2.1 代码拉取
从 Git 仓库拉取最新代码：
git clone https://github.com/company/product.git
cd product
git checkout main

2.2 配置文件
复制环境变量模板并修改：
cp .env.example .env

需要配置的关键参数：
- DATABASE_URL：数据库连接字符串
- REDIS_URL：Redis 连接字符串
- SECRET_KEY：应用密钥
- API_KEY：第三方服务密钥
- LOG_LEVEL：日志级别

2.3 数据库初始化
执行数据库迁移：
python manage.py migrate

加载初始数据：
python manage.py loaddata fixtures/initial_data.json

2.4 启动服务
使用 Docker Compose 启动所有服务：
docker-compose up -d

验证服务状态：
docker-compose ps
curl http://localhost:8080/health

第三章 监控与日志

3.1 监控配置
Prometheus 配置：
- 采集间隔：15秒
- 保留时间：30天
- 告警规则：CPU > 80%，内存 > 85%，磁盘 > 90%

Grafana 仪表盘：
- 应用性能指标
- 数据库性能指标
- 系统资源指标
- 业务指标

3.2 日志管理
日志级别：
- DEBUG：开发环境
- INFO：生产环境
- WARNING：异常但可恢复
- ERROR：需要立即处理

日志轮转：
- 单个日志文件最大 100MB
- 保留最近 30 天的日志
- 压缩历史日志

第四章 备份与恢复

4.1 数据库备份
每日自动备份：
- 备份时间：凌晨 2:00
- 备份保留：最近 7 天
- 异地备份：每周同步到云端存储

手动备份命令：
pg_dump -h localhost -U postgres product_db > backup.sql

4.2 数据恢复
恢复数据库：
psql -h localhost -U postgres product_db < backup.sql

恢复后验证：
- 检查表完整性
- 验证关键数据
- 测试应用功能

第五章 故障排查

5.1 常见问题
服务无法启动：
- 检查端口占用
- 检查磁盘空间
- 检查配置文件
- 查看启动日志

数据库连接失败：
- 检查数据库服务状态
- 验证连接字符串
- 检查防火墙规则
- 查看数据库日志

5.2 紧急处理
服务宕机：
1. 检查服务状态
2. 查看错误日志
3. 尝试重启服务
4. 联系技术负责人

数据异常：
1. 立即停止写入
2. 备份当前数据
3. 分析异常原因
4. 执行数据修复
